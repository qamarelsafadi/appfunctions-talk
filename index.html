// build.gradle.kts plugins { id("com.google.devtools.ksp") } dependencies { // Core AppFunctions API implementation("androidx.appfunctions:appfunctions:1.0.0-alpha01") // Service for handling agent execution implementation("androidx.appfunctions:appfunctions-service:1.0.0-alpha01") // KSP compiler — generates metadata ksp("androidx.appfunctions:appfunctions-compiler:1.0.0-alpha01") } // KSP config ksp { arg("aggregateAppFunctions", "true") }

💡 aggregateAppFunctions = true ensures all functions across modules are collected into a single metadata file

Code

Defining an AppFunction

/** * Find people or groups by name, phone number, or email. * * MUST be called before [sendMessage] or [makeCall] * to resolve the correct contact endpoint. * * @param query the name, number, or email to search for * @return list of matching contacts with endpoint values */ @AppFunction(isDescribedByKDoc = true) suspend fun searchContacts( appFunctionContext: AppFunctionContext, query: String ): List<ContactSearchResult>

📖 KDoc is part of the contract — Gemini reads it to understand how to use this function

Best Practice

KDoc = Agent Behavior

The quality of your KDoc directly controls how well the agent uses your function

❌ Vague
/**
 * Searches contacts.
 */

Agent doesn't know when to call this, what params to use, or what comes next

✅ Descriptive
/**
 * Find people or groups by name.
 * MUST be called before
 * [sendMessage] or [makeCall]
 * to resolve the correct
 * contact endpoint.
 *
 * @param query the name, number,
 * or email to search for
 * @return list of matching contacts
 * with endpoint values
 */

KDoc = the agent instruction layer

The Big Picture

AppFunctions ≈ MCP for Your App

🌐 MCP (Model Context Protocol)
• Server exposes tools
• Tools have schemas + descriptions
• LLM discovers & invokes tools
• Works across network boundaries
≈
📱 AppFunctions
• App exposes @AppFunctions
• KDoc + types = self-describing
• Agent discovers & invokes functions
• Works on-device via Android system

MCP lets LLMs talk to servers. AppFunctions let agents talk to Android apps.
Same philosophy. Native on mobile.

Takeaways

Best Practices

📖
KDoc Is the Contract
Your KDoc is what the agent reads to understand function behavior, parameters, ordering, and edge cases. Write it like API documentation for an LLM.
🧩
Design Composable Functions
Build small, focused functions that chain together. Let the agent orchestrate the workflow instead of building monolithic endpoints.
⚠️
Make Errors Agent-Readable
Return structured error messages the agent can interpret and recover from. Don't throw exceptions — return error states in your result types.
🧪
Test Early with ADB
Use ADB commands and AppFunctionStudio to verify your functions are properly registered and discoverable before waiting for agent integration.
